# Naming Conventions

Use clear, reusable file names for Wiki notes.

- `Wiki/Topics/` for broad subject overviews
- `Wiki/Concepts/` for discrete ideas and definitions
- `Wiki/Entities/` for people, organizations, tools, or objects
- `Wiki/Projects/` for workflows, plans, or initiatives
- `Wiki/Logs/` for maintenance and audit history

File names should be:

- short and descriptive
- title-case or sentence-case
- lowercase only when useful for compatibility
- free of spaces when a link-friendly slug is needed
